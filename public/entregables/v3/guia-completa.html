Que es Claude Code y por que es diferente

Claude Code no es un chatbot. Es un agente de software con acceso completo a tu terminal, sistema de archivos y herramientas de desarrollo.

  • Es proactivo: no espera instrucciones paso a paso, resuelve problemas
  • Tiene acceso total: lee y escribe archivos, ejecuta comandos, navega tu proyecto
  • La clave es el contexto: dale informacion, no direccion rigida
Cambio de mentalidad

Con Claude Chat le dices que hacer. Con Claude Code le dices que quieres lograr y el decide como hacerlo.

El archivo CLAUDE.md: tu manual de instrucciones

El CLAUDE.md es el archivo mas importante de cualquier proyecto con Claude Code. Tiene 4 niveles:

NivelUbicacionAlcance
Global~/.claude/CLAUDE.mdAplica a TODO lo que hagas
ProyectoCLAUDE.md en la raizSolo este proyecto
SubcarpetaCLAUDE.md en una carpetaSolo esa carpeta
TemporalInstrucciones en el chatSolo esta conversacion
Error comun

No metas 500 lineas en el CLAUDE.md. Debe ser un diseminador (50 lineas maximo) que apunte a donde buscar mas informacion. Si contiene todo, consume tokens innecesariamente.

4

La entrevista de 7 ambitos

Antes de empezar cualquier proyecto, haz que Claude te interrogue sobre 7 areas clave:

  1. Identidad profesional: que haces, para quien, experiencia
  2. Equipo y contexto: trabajo solo, en equipo, jerarquia
  3. Proyectos activos: en que estas trabajando ahora
  4. Casos de uso con IA: para que usas o quieres usar la IA
  5. Preferencias de comunicacion: directo, detallado, tecnico, simple
  6. Mundo personal: hobbies, intereses que influyen en tu trabajo
  7. Limites: que NO quieres que haga, temas sensibles
Por que funciona

Cuanto mas te conozca Claude, mejor seran sus respuestas. La entrevista de 7 ambitos construye un perfil completo que persiste en tu CLAUDE.md global.

5

Pipeline de despliegue: GitHub + Vercel

El flujo para tener una web en produccion sin saber programar:

  1. Escribes codigo con Claude Code (el lo genera, tu lo diriges)
  2. Subes a GitHub con git push
  3. Conectas con Vercel (una sola vez)
  4. Deploy automatico con cada push
Resultado

Cada vez que haces git push, tu web se actualiza automaticamente. Sin servidores, sin configuracion manual, sin saber DevOps.

6

La IA no sustituye aprender

Uno de los hallazgos mas importantes del curso: la IA no sustituye aprender, te ayuda a aprender mejor.

El ejemplo de AIM (diagnostico medico):

  • IA sola: diagnostica mejor que medicos solos
  • Medicos + IA: ligeramente mejor que IA sola
  • Conclusion: el humano aporta criterio, no velocidad
La leccion

Si no aprendes el dominio, te conviertes en el lastre del sistema. La IA te hace mas rapido, pero tu criterio es lo que la hace util.

Prompts de esta seccion

Prompt 1: Entrevista de 7 ambitos

Prompt copiable
Quiero que me hagas una entrevista exhaustiva para conocerme y poder trabajar conmigo de forma efectiva. Cubre estos 7 ambitos uno por uno: 1. Identidad profesional (que hago, para quien, experiencia) 2. Equipo y contexto (trabajo solo, en equipo, jerarquia) 3. Proyectos activos (en que estoy trabajando ahora) 4. Casos de uso con IA (para que uso o quiero usar la IA) 5. Preferencias de comunicacion (directo, detallado, tecnico, simple) 6. Mundo personal (hobbies, intereses que influyen en mi trabajo) 7. Limites (que NO quiero que hagas, temas sensibles) Hazme las preguntas una por una. Al final, genera un perfil profesional completo.

Prompt 2: Crear CLAUDE.md

Prompt copiable
Analiza este proyecto y genera un archivo CLAUDE.md conciso (maximo 50 lineas) que incluya: - What: que es este proyecto en 2 frases - Stack: tecnologias usadas - Structure: carpetas clave y su proposito - Rules: reglas no negociables del proyecto - Conventions: convenciones de codigo No metas contenido que pueda encontrarse en otros archivos. El CLAUDE.md es un diseminador que apunta a donde buscar, no un contenedor de todo.

Prompt 3: Delegar con criterio

Prompt copiable
Para esta tarea: [describe la tarea] Disena el equipo de agentes que consideres necesario. Dame 3 opciones: 1) La mas economica (menos tokens, mas rapida) 2) La mas funcional (mejor resultado) 3) La mas segura (con mas verificaciones) Para cada opcion indica: agentes necesarios, flujo de trabajo, tokens estimados, riesgos.

Resumen: lo que debes recordar

  1. Claude Code es un agente con acceso total a tu sistema — no un chat
  2. 5 pilares: CLAUDE.md, permisos, hooks, MCPs, subagentes
  3. CLAUDE.md conciso (50 lineas) — apunta a donde buscar, no contiene todo
  4. Entrevista de 7 ambitos obligatoria antes de empezar cualquier proyecto
  5. GitHub + Vercel = deploy automatico con cada git push
  6. La IA no sustituye aprender — amplifica tu capacidad de aprendizaje

Tarea practica del modulo

  1. Instala Claude Code siguiendo el checklist de setup
  2. Ejecuta la entrevista de 7 ambitos con el Prompt 1
  3. Crea tu primer CLAUDE.md global en ~/.claude/CLAUDE.md con el Prompt 2
  4. Conecta un proyecto a GitHub y Vercel para deploy automatico
  5. Prueba a pedirle a Claude Code que audite un proyecto existente tuyo
