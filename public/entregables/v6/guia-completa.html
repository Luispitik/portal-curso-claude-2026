Fase 0: Pensar antes de construir

Antes de escribir una sola linea de codigo, necesitas pensar:

  • Que quieres construir: define el producto en 2 frases
  • Para quien: quien va a usar esto y que necesita
  • Que NO lleva: lo que excluyes es tan importante como lo que incluyes
Regla de oro

Un prompt bien pensado de 300 lineas te ahorra 3 horas de ida y vuelta con Claude. Invierte en el prompt, no en las correcciones.

2

Fase 1: El prompt maestro

La estructura de un prompt maestro para un proyecto completo:

  1. Que es esto: descripcion del producto
  2. Stack: tecnologias a usar
  3. Lo que NO lleva: exclusiones explicitas
  4. Estructura de carpetas: arbol de archivos completo
  5. Design tokens: colores, fuentes, spacing
  6. Datos: contenido real, no lorem ipsum
  7. Plan de ejecucion: fases ordenadas con commits
Ejemplo real

El prompt que construyo este portal tiene 300+ lineas. Incluye stack, estructura, design tokens, datos de las 6 secciones y plan de ejecucion en 6 fases. Esta disponible como entregable descargable.

3

Fase 2: MCP en accion

En la sesion en vivo usamos estos conectores MCP:

  • Context7: documentacion de librerias actualizada en tiempo real
  • Excalidraw: diagramas y esquemas visuales
  • Vercel: deploy y preview automatico
  • Google Workspace: docs, calendar, drive integrados
Clave

Los MCPs no son "extras" — son lo que convierte a Claude de un chatbot a un sistema operativo que se conecta a tus herramientas reales.

4

Fase 3: Vibecoding real

Vibecoding no es "dile a la IA que haga todo". Es colaboracion en tiempo real:

  • Tu diriges: defines que quieres y por que
  • Claude ejecuta: genera codigo, archivos, configuraciones
  • Tu verificas: revisas el resultado, das feedback
  • Iterais juntos: ajustais hasta que quede bien
Error comun

El mayor error del vibecoding es aceptar todo sin revisar. Recuerda el modulo V2: activa tu Sistema 2 y aplica criterio a cada output.

5

Fase 4: Deploy con Vercel

El flujo completo de deploy:

  1. git add . + git commit — preparar cambios
  2. git push — subir a GitHub
  3. Vercel detecta el push y despliega automaticamente
  4. Variables de entorno en el dashboard de Vercel
  5. Preview deployments para cada Pull Request
6

Fase 5: De proyecto a skill

Todo proyecto exitoso puede convertirse en una skill reutilizable:

  • El patron de proyecto que seguiste
  • Las decisiones arquitectonicas que funcionaron
  • Los gotchas que encontraste
  • Un template de prompt maestro para proyectos similares
  • Un checklist de verificacion pre-deploy
El circulo completo

Este portal de entregables se convierte en un template reutilizable. El proximo curso que hagas, clonas la estructura y solo cambias el contenido. De proyecto a skill, de skill a sistema.

Prompts de esta seccion

Prompt 1: Estructurar un proyecto nuevo

Prompt copiable
Voy a construir [describe el proyecto]. Necesito que me ayudes a crear un prompt maestro que incluya: 1. Que es y para quien 2. Stack tecnologico recomendado 3. Lo que NO debe llevar (mantenerlo simple) 4. Estructura de carpetas 5. Design tokens (colores, fuentes) 6. Plan de ejecucion por fases Hazme preguntas si necesitas mas contexto.

Prompt 2: Convertir proyecto en skill

Prompt copiable
Acabamos de completar [describe proyecto]. Quiero convertir lo aprendido en una skill reutilizable. Genera un SKILL.md que incluya: 1. El patron de proyecto que seguimos 2. Las decisiones arquitectonicas que funcionaron 3. Los gotchas que encontramos 4. Un template de prompt maestro para proyectos similares 5. Checklist de verificacion pre-deploy

Prompt 3: Auditar antes de deploy

Prompt copiable
Audita este proyecto antes de desplegarlo a produccion. Revisa: 1. Seguridad (secrets, env vars, headers) 2. SEO (meta tags, sitemap, robots) 3. Performance (bundle size, imagenes, caching) 4. Accesibilidad basica (alt texts, contraste, focus) 5. Contenido (links rotos, typos, placeholder text) Dame un informe con severidad (critico/alto/medio/bajo) y fix para cada hallazgo.

Resumen: lo que debes recordar

  1. Pensar antes de construir — un prompt de 300 lineas ahorra 3 horas
  2. Los MCPs convierten Claude en un sistema operativo conectado
  3. Vibecoding es colaborar, no delegar ciegamente
  4. git push = deploy automatico con Vercel
  5. Todo proyecto exitoso puede ser skill — cierra el circulo
