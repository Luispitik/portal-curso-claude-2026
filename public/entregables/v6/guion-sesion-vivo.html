Sistema de contenido MDX

El motor de auto-publicacion
8 min
"Ahora viene lo interesante. Vamos a hacer que el contenido se publique solo. Sin tocar codigo. Solo creando un archivo."
Ahora crea el sistema de contenido MDX: 1. Un loader en src/lib/sections.ts que lea archivos .mdx de src/content/secciones/, parsee frontmatter con gray-matter, y retorne la lista de secciones 2. Los archivos .mdx con este frontmatter: title, slug, number, subtitle, description, status, duration, entregables (array), keyTakeaways (array), tarea 3. Crea los 5 archivos MDX para los 5 videos: V1 Bienvenida, V2 Framework Mental, V3 Claude Code, V4 Skills/Hooks/Plugins, V5 Synapsis 4. Conecta el loader con las paginas (home + detalle) 5. Los entregables de cada seccion deben enlazar a archivos en public/entregables/v1/, v2/, etc.

Explicar mientras Claude construye

"Lo que esta haciendo ahora es crear archivos .mdx que son como Markdown con superpoderes. Tienen un frontmatter arriba con los datos y abajo el contenido. Cuando yo anada un archivo nuevo aqui y haga push, aparece en la web automaticamente. Sin tocar nada de codigo."
TERCER COMMIT: "feat: MDX content system with 5 sections"

API tracking de descargas + Supabase

Registro de quien descarga que + seguridad Conecta con V1+V3+V4
8 min
"Los materiales se descargan directamente, sin barreras. Pero quiero saber que se descarga y cuanto. Vamos a conectar Supabase para registrar cada descarga — sin bloquear nada."

Paso 1: Crear tabla en Supabase

"Abro Supabase, creo una tabla 'downloads' con cuatro campos: file, section, downloaded_at y user_agent. Tarda 30 segundos."
Crea el sistema de tracking de descargas: 1. src/lib/supabase.ts — client singleton (SUPABASE_URL + SERVICE_ROLE_KEY) 2. /api/track-download (POST): - Recibe: { file, section } - Guarda en Supabase tabla "downloads": file, section, downloaded_at (now), user_agent - Rate limit: 10/IP/min - Si falla, responde 200 igual (no bloquear la descarga) 3. Componente DownloadButton: - Abre el archivo directamente (nueva pestana para HTML, descarga para MD) - En paralelo, fetch a /api/track-download (fire-and-forget) - Iconos diferentes segun tipo (HTML/MD/JSON) - Animacion sutil al click 4. .env.example con SUPABASE_URL y SUPABASE_SERVICE_ROLE_KEY

Explicar mientras se construye

"Fijaos: el boton abre el archivo directamente. No hay barrera. Pero en paralelo, en background, registra la descarga en Supabase. Si Supabase se cae, no pasa nada — la descarga ya se hizo. Es tracking sin friccion."
"Esto es exactamente lo que hacia con N8N pero nativo. Sin servidor externo, sin webhook, sin nada. Una API route de 20 lineas conectada a Supabase."
CUARTO COMMIT: "feat: download tracking with Supabase"

Convertir el proyecto en Skill

Cerrar el ciclo Conecta con V4+V5
8 min
"Recordais la maxima de Synapsis? No repetir lo mismo dos veces. Acabamos de construir un portal. Si manana quiero hacer otro portal similar para otro curso, no quiero repetir todo esto. Voy a crear una skill."

Paso 1: Analizar patrones

Analiza lo que acabamos de construir y dime: 1. Que patrones se repitieron (stack, estructura, decisiones) 2. Que preguntas de la entrevista serian utiles para cualquier portal de contenido similar 3. Que partes se podrian reutilizar como template

Paso 2: Crear la skill

Crea una skill reutilizable: Nombre: portal-contenido-mdx Proposito: Crear portales web de contenido organizado con MDX auto-publicable para cursos, comunidades o documentacion. Incluye: 1. Stack fijo y dependencias 2. Preguntas de entrevista al usuario 3. Estructura de proyecto estandar 4. Proceso paso a paso 5. Los gotchas que encontramos durante el build Guardala en ~/.claude/skills/_library/salgadoia-portal-contenido/SKILL.md
"Veis? Lo que nos llevo 30 minutos ahora esta documentado como skill. La proxima vez que alguien diga 'quiero un portal de contenido', invoco esta skill y el proceso esta sistematizado. Esto es el ciclo completo: proyecto → patrones → skill → sistema."
