Synapsis 3.2: Tu Sistema de Aprendizaje Continuo

El sistema que observa como trabajas, detecta patrones y convierte tu experiencia en procedimientos reutilizables. Automaticamente.

1 Que es Synapsis (y que NO es)

En la seccion 5 viste que Synapsis detecta patrones y crea instincts. Hoy vamos a abrir el capo y ver como funciona por dentro, que archivos lo componen, y como operarlo dia a dia.

En una frase: Synapsis es un observador pasivo que mira cada sesion de trabajo, captura los patrones que repites, les asigna un nivel de confianza, y cuando estan maduros los convierte en skills o reglas que Claude aplica automaticamente.

Lo que SI es

  • Un sistema de aprendizaje continuo que mejora con cada sesion
  • Un observador pasivo — no interrumpe, no pregunta, solo mira
  • Un generador de skills a partir de patrones reales de trabajo
  • Un cerebro operativo que recuerda decisiones entre proyectos

Lo que NO es

  • NO es magia — necesita 2-3 sesiones minimo para empezar a detectar patrones
  • NO toma decisiones por ti — sugiere, tu apruebas
  • NO funciona offline — necesita que trabajes con Claude para observar
  • NO es un plugin externo — esta integrado en tu sistema de skills

2 Arquitectura: las 4 capas del sistema

Synapsis no es un solo archivo. Es un sistema de 4 capas que trabajan juntas. Cada capa tiene su archivo y su funcion:

Capa 1: Memoria Operativa (Operator State)

Las decisiones estrategicas que aplican a TODOS tus proyectos: tu stack, tech retirada, lecciones aprendidas, milestones pendientes. Claude lo lee al iniciar CADA sesion.

~/.claude/skills/_operator-state.json

Capa 2: Skill Router (Orquestador)

El cerebro que detecta que tipo de proyecto tienes, recomienda skills, las instala, y captura decisiones nuevas. Es la interfaz entre tu y el sistema.

~/.claude/skills/skill-router/SKILL.md

Capa 3: Biblioteca + Catalogo + Reglas

La coleccion de skills dormantes (se activan por proyecto), el indice para buscarlas, los proyectos registrados, y las reglas pasivas que se aplican solas.

~/.claude/skills/_library/ + _catalog.json + _projects.json + _passive-rules.json

Capa 4: Aprendizaje Continuo (Synapsis)

El observador que captura patrones, los convierte en instincts con confidence score, y los evoluciona en skills cuando maduran.

~/.claude/skills/norteia-continuous-learning/SKILL.md

Analogia: Es como una empresa. La Capa 1 es la direccion (decisiones estrategicas). La Capa 2 es el manager (asigna recursos). La Capa 3 es el almacen (herramientas disponibles). La Capa 4 es el becario observador que documenta procesos.

3 El ciclo de vida: de observacion a skill

Este es el flujo completo. Cada patron que repites sigue este camino:

1

Observacion

Tu trabajas normalmente. Synapsis observa cada accion: que archivos editas, que comandos usas, que patrones sigues, como corriges a Claude. No interfiere. Solo mira.

2

Deteccion de patron

Despues de 2-3 sesiones, Synapsis detecta repeticiones: "este usuario siempre pide conventional commits", "siempre usa Tailwind con shadcn", "siempre verifica auth en API routes".

3

Instinct (patron atomico)

El patron se guarda como "instinct": una regla atomica con nombre, descripcion, y confidence score de 0 a 1. Cuantas mas veces se repite, mas sube la confianza.

4

Maduracion

El instinct acumula evidencia. Si el usuario confirma el patron (+0.05), sube. Si lo contradice (-0.10), baja. Cuando llega a 0.7+, esta listo para evolucionar.

5

Evolucion → Skill / Regla pasiva

Con /evolve, Synapsis agrupa instincts relacionados y propone: convertirlos en una skill (procedimiento completo), una regla pasiva (se aplica sola), o un comando (accion invocable).

Confidence score: como madura un instinct

ObservacionesConfidenceEstadoEjemplo
1-20.30 Semilla — demasiado pronto para actuar
3-50.50 Tendencia — parece un patron pero puede ser coincidencia
6-100.70 Patron — listo para evolucionar si quieres
11+0.85+ Validado — Synapsis lo aplica o sugiere con alta confianza

Dato real: En mi sistema, "api-jwt-auth-required" tiene confidence 0.95. Eso significa que en practicamente todos mis proyectos, Claude aplica autenticacion JWT en API routes automaticamente, sin que yo se lo pida. Porque Synapsis lo aprendio observandome.

_catalog.json — El indice de skills

Un JSON con las 41 skills disponibles. Cada skill tiene:

{ "id": "salgadoia-nextjs-pro", "name": "Next.js Pro Patterns", "description": "Next.js 16, Server Components, Tailwind, Vercel", "tags": ["nextjs", "react", "tailwind", "vercel"], "triggers": ["crear web", "next.js", "landing"], "category": "web-dev" }

Los triggers son las palabras clave que el router usa para autodetectar que skill recomendar cuando describes una tarea.

_projects.json — Tus proyectos conocidos

Registro de todos los proyectos con los que has trabajado. Incluye:

  • Historial de clonaciones (clonedFrom) para heredar configuracion

5 El Skill Router: tu launcher inteligente

El router es la puerta de entrada al sistema. Cuando dices "launcher" o "skills", se activa y te ofrece:

Tienes 3 opciones: [1] Skills on Demand — Launcher con proyectos y skills → Detecta tu proyecto, sugiere skills, instala lo que necesitas [2] Skill Picker — Elige skills manualmente → Te muestra el catalogo completo por categorias [3] Freestyle — Claude vanilla → Sin sistema, sin skills, sin reglas

Si eliges [1] Launcher con proyecto existente:

  1. 1Detecta CLAUDE.md en tu carpeta → sabe que proyecto es
  2. 2Hace inventario de skills instaladas en .claude/commands/
  3. 3Cruza con Synapsis: hay instincts nuevos relevantes?
  4. 4Te da un "Skill Fitness Report": que tienes, que te falta, que sugiere
  5. 5Tu decides: instalar sugeridas, actualizar una, limpiar las que no usas, o seguir como esta

6 Los 10 comandos que necesitas conocer

Estos son los comandos para operar Synapsis dia a dia. No necesitas memorizarlos todos, pero si saber que existen.

/system-status

Dashboard completo: skills activas, instincts, reglas pasivas, observaciones, proyectos. Tu "cuadro de mando".

/instinct-status

Lista todos los instincts detectados con su confidence score, scope y veces detectado.

/evolve

Agrupa instincts maduros y propone crear skills, reglas o comandos nuevos. El "upgrade" del sistema.

/promote

Sube un instinct de scope proyecto a scope global. Aplica a TODOS tus proyectos.

/passive-status

Muestra las 16 reglas pasivas activas, cuales se activan mas y cuales nunca se han usado.

/analyze-observations

Analiza las observaciones de Synapsis y sugiere instincts, skills y reglas nuevas.

/projects

Lista proyectos conocidos con conteo de instincts y observaciones por proyecto.

/session-end

Cierra sesion documentando lo hecho, lo pendiente y las decisiones. Guarda en memoria.

/instinct-export

Exporta instincts a fichero para backup o compartir con otro sistema.

/instinct-import

Importa instincts desde fichero con control de scope (proyecto o global).

7 Que esperar: timeline realista

Synapsis no da resultados inmediatos. Necesita observarte trabajar. Este es el timeline real:

CuandoQue pasaQue haces tu
Semana 1 Synapsis observa. Captura primeras observaciones. Ningun instinct todavia. Trabaja normalmente. No fuerces nada.
Semana 2 5-10 instincts detectados, la mayoria con confidence baja (0.3-0.5). Ejecuta /instinct-status para ver que ha detectado.
Semana 3 Instincts maduran. Algunos llegan a 0.7+. /evolve empieza a sugerir. Prueba /evolve. Decide si las sugerencias tienen sentido.
Semana 4 2-3 skills auto-generadas. Primeras reglas pasivas activas. Valida: las reglas aplican? Las skills funcionan?
Mes 2+ Sistema maduro. El /system-status muestra un ecosistema real. Crece organicamente. Cada proyecto nuevo hereda el sistema.

Error comun: "Instale Synapsis hace 2 dias y no tiene instincts." Normal. Necesita 6-10 observaciones del mismo patron para crear un instinct con confianza minima. Trabaja 2-3 sesiones reales y vuelve a mirar.
