Que es Synapsis

Synapsis es un sistema de aprendizaje continuo. Es un observador pasivo que captura patrones de tu trabajo con Claude y los convierte en conocimiento reutilizable.

Principio fundamental

Nunca repitas la misma correccion dos veces. Si corriges a Claude, Synapsis lo captura y lo aplica automaticamente la proxima vez.

3

Ciclo de vida de un instinct

Todo empieza con una observacion y termina en una skill permanente:

  1. Observacion: Claude detecta un patron en tu trabajo
  2. Captura: Lo registra como instinct con confidence 0.30 (semilla)
  3. Repeticion: Cada vez que el patron se repite, sube el confidence
  4. Maduracion: Cuando llega a 0.90+, es candidato a evolucion
  5. Evolucion: /evolve lo convierte en skill, regla pasiva o hook

Reglas pasivas y Operator State

Reglas pasivas: se aplican silenciosamente sin que tengas que pedirlo.

  • "Nunca commitear .env" — se activa cada git commit
  • "Colores hex 6 digitos en docx" — se activa al generar documentos
  • "Build lint antes de commit" — pipeline automatico

Operator State: decisiones estrategicas que persisten entre sesiones y proyectos.

  • "n8n retirado del stack activo" — advierte si un proyecto lo usa
  • "HTML twin en cada entregable" — genera version HTML de todo documento
  • "Nuevo proyecto requiere monetizacion" — pregunta antes de crear

Prompts de esta seccion

Prompt 1: Activar modo observacion

Prompt copiable
Quiero que actues como un sistema de aprendizaje continuo. A partir de ahora: 1. Observa patrones en como trabajo y las correcciones que te hago 2. Cuando detectes un patron que se repite 2+ veces, mencionalo 3. Sugiere convertirlo en una regla permanente 4. Mantene un registro mental de mis preferencias No necesitas hacer nada ahora — solo observa y aprende.

Prompt 2: Revisar instincts

Prompt copiable
Revisa todos los patrones que has observado en nuestras conversaciones. Para cada uno: 1. Describe el patron detectado 2. Cuantas veces lo has visto (frecuencia) 3. Nivel de confianza (0-1) 4. Propuesta: convertir en regla, skill, o ignorar Ordenados de mayor a menor confianza.

Prompt 3: Evolucionar un patron

Prompt copiable
El patron [describe patron] se ha repetido suficientes veces. Quiero que lo conviertas en: - Opcion A: una regla pasiva (se aplica siempre sin preguntar) - Opcion B: una skill completa (archivo SKILL.md) - Opcion C: un hook automatico Generame las 3 opciones y recomienda cual es mejor para este caso.

Resumen: lo que debes recordar

  1. 4 capas: Operator State, Skill Router, Biblioteca, Aprendizaje Continuo
  2. Confidence scoring: de semilla (0.30) a validado (0.90+)
  3. /evolve convierte patrones maduros en skills permanentes
  4. Nunca repitas la misma correccion dos veces
  5. Las reglas pasivas trabajan en silencio por ti

Tarea practica del modulo

  1. Usa el Prompt 1 para activar el modo observacion
  2. Trabaja normalmente durante 3-5 sesiones
  3. Usa el Prompt 2 para revisar que ha aprendido Claude
  4. Elige un patron maduro y usa el Prompt 3 para evolucionarlo
  5. Revisa el entregable "Synapsis Detalle Sesion A" para ver un ejemplo real
