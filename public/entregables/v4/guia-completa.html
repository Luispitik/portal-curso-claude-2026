2

Skills: la receta reutilizable

Una skill es un archivo SKILL.md con instrucciones que Claude lee y ejecuta. Es como una receta de cocina: ingredientes (contexto), pasos (proceso), resultado esperado (output).

  • Tiene frontmatter YAML: nombre, descripcion, triggers, tags
  • Se guarda en .claude/commands/ para convertirse en slash command
  • Ejemplos reales: Business Launcher, Deep Research, Word Creator, Security Shield
Regla de oro

Si repites una tarea mas de 2 veces, conviertela en skill. Ejemplo: investigar un lead siempre sigue los mismos pasos -> skill Lead Research.

3

Hooks: la campana automatica

Un hook es un trigger automatico que se dispara cuando ocurre un evento. Se configuran en settings.json.

Tipo de HookCuando se disparaEjemplo
PreToolUseAntes de usar una herramientaVerificar que no vas a commitear .env
PostToolUseDespues de usar una herramientaLint automatico despues de escribir codigo
NotificationCuando hay un avisoAlerta de seguridad si detecta un secret
4

Plugins: la franquicia completa

Un plugin empaqueta skills + hooks + MCPs + agentes en un sistema distribuible. Analogia: si la skill es una receta y el hook es una campana, el plugin es la franquicia — todo empaquetado para replicar.

Ejemplo real

Synapsis 3.2 es un plugin con: skill de observacion, hook de captura de instincts, MCP de contexto, y subagentes de evolucion. Todo junto, instalable en cualquier proyecto.

5

Como crear tu primera skill

Paso a paso:

  1. Identifica una tarea que repitas regularmente
  2. Escribe las instrucciones como si se las dieras a una persona
  3. Estructura en SKILL.md con frontmatter YAML
  4. Guarda en .claude/commands/
  5. Prueba con /tu-skill como slash command

Template basico de SKILL.md:

Template copiable
--- name: mi-skill description: "Que hace esta skill en una linea" triggers: - "cuando el usuario dice X" - "cuando se detecta Y" tags: - tag1 - tag2 --- # Mi Skill ## Cuando activar - Situacion 1 - Situacion 2 ## Reglas 1. Regla no negociable 1 2. Regla no negociable 2 ## Proceso 1. Paso 1 2. Paso 2 3. Paso 3 ## Output esperado - Que debe generar esta skill
6

Flujo seguro: nunca a main

Regla absoluta: nunca hagas push directo a main.

  1. git checkout -b feat/mi-feature — crea rama
  2. Trabaja y haz commits
  3. git push origin feat/mi-feature — sube la rama
  4. Abre un Pull Request
  5. Revisa (o pide a Claude que revise)
  6. Merge a main
Por que importa

Si rompes main, rompes produccion. Las ramas son tu red de seguridad. Incluso trabajando solo, este habito te salvara.

Prompts de esta seccion

Prompt 1: Identificar skills potenciales

Prompt copiable
Analiza mi flujo de trabajo reciente. Identificame las 5 tareas que mas repito y que podrian convertirse en skills reutilizables. Para cada una, dame: 1. Nombre propuesto para la skill 2. Que hace en una frase 3. Triggers (cuando se activa) 4. Complejidad estimada (baja/media/alta)

Prompt 2: Crear una skill desde cero

Prompt copiable
Quiero crear una skill para [describe la tarea]. Genera el archivo SKILL.md completo con: - Frontmatter YAML (name, description, triggers, tags) - Seccion "Cuando activar" - Reglas no negociables - Proceso paso a paso - Output esperado - Ejemplos de uso Hazla profesional y lista para usar como slash command.

Prompt 3: Configurar un hook

Prompt copiable
Necesito un hook que se active cuando [describe el evento]. El hook debe [describe la accion]. Generame: 1. La configuracion para settings.json 2. El script del hook si es necesario 3. Instrucciones de instalacion paso a paso 4. Como verificar que funciona

Resumen: lo que debes recordar

  1. Skills = recetas, Hooks = campanas, Plugins = franquicias
  2. 4 niveles de CLAUDE.md: global, proyecto, subcarpeta, temporal
  3. Si repites algo 2+ veces, conviertelo en skill
  4. Hooks automatizan en settings.json con triggers de eventos
  5. Nunca push a main — siempre rama + review

Tarea practica del modulo

  1. Identifica 3 tareas que repites usando el Prompt 1
  2. Crea tu primera skill con el Prompt 2 y guardala en .claude/commands/
  3. Prueba a invocarla como slash command
  4. Revisa el ejemplo-skill-whisper.md descargable de esta seccion
  5. Crea una rama nueva y practica el flujo: branch, commit, push, PR
