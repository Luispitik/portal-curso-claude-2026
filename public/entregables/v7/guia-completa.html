Que son las tareas programadas

Las tareas programadas de Claude Code son agentes autonomos que ejecutan trabajo en segundo plano segun un horario que tu defines. Se configuran desde la seccion "Programar" del panel lateral de Claude Code.

  • Se ejecutan solo con el ordenador encendido
  • Si el ordenador estaba apagado, se recuperan al encenderlo
  • La primera vez piden permisos (puedes ponerlas en modo bypass)
  • Consumen tokens — son agentes reales, no simples scripts
Diferencia clave

Las tareas programadas son como tener un empleado que viene cada dia a una hora concreta, hace su trabajo y se va. Tu solo revisas el resultado.

3

Como configurarlas

  1. Abre Claude Code y ve a la seccion "Programar" en el panel lateral
  2. Crea una nueva tarea con un nombre descriptivo
  3. Define el prompt (que quieres que haga el agente)
  4. Configura la frecuencia (diario, semanal, cada X horas)
  5. La primera vez, acepta los permisos
  6. Opcionalmente, pon en modo bypass para que no pregunte
Cuidado

El modo bypass significa que la tarea se ejecuta sin supervision. Solo activalo en tareas que hayas verificado que funcionan bien manualmente primero.

Tareas vs Hooks: cuando usar cada uno

CaracteristicaTareas programadasHooks
TriggerHorario (cron)Evento (pre/post tool use)
Consumo tokensSi (son agentes)No necesariamente
ComplejidadAlta (trabajo completo)Baja (automatizacion puntual)
EjemploGenera newsletter semanalLint automatico al guardar
AnalogiaEmpleado con horarioFlujo automatico (como n8n)
La combinacion ganadora

Las skills definen QUE hacer, los hooks disparan CUANDO hacerlo, y las tareas programadas lo ejecutan AUTOMATICAMENTE. Juntos, forman un sistema operativo completo.

5

Combinando tareas + hooks + skills

El flujo completo para automatizar un proceso de negocio:

  1. Identifica un proceso repetitivo (ej: follow-up de leads)
  2. Crea la skill que define como hacerlo (lead-follow-up.md)
  3. Configura un hook que la active ante ciertos eventos (nuevo email de lead)
  4. Programa una tarea que la ejecute periodicamente (cada viernes, revisa pendientes)
  5. Verifica manualmente las primeras ejecuciones
  6. Activa bypass cuando estes seguro de que funciona

Prompts de esta seccion

Prompt 1: Identificar automatizaciones

Prompt copiable
Analiza mi flujo de trabajo semanal. Quiero identificar que tareas podria automatizar con tareas programadas de Claude Code. Para cada una, dame: 1. Nombre de la tarea 2. Que hace exactamente 3. Frecuencia recomendada (diaria, semanal, cada X horas) 4. Skills necesarias 5. Riesgo de automatizar sin supervision (bajo/medio/alto) Empieza preguntandome sobre mis procesos repetitivos.

Prompt 2: Crear tarea programada

Prompt copiable
Quiero crear una tarea programada que [describe la tarea]. Necesito: 1. El prompt completo para el agente 2. La frecuencia optima (con expresion cron) 3. Las skills que deberia tener instaladas 4. Un checklist de verificacion antes de activar bypass 5. Que podria fallar y como detectarlo

Prompt 3: Resumen semanal automatico

Prompt copiable
Eres un agente de resumen semanal. Tu tarea es: 1. Revisa las sesiones de Claude Code de esta semana 2. Identifica los 5 hallazgos mas importantes 3. Genera un articulo de 500 palabras maximo con los descubrimientos 4. Formatealo como borrador de newsletter/blog 5. Enviamelo como borrador de email a [tu email] Tono: practico, directo, con ejemplos concretos.

Resumen: lo que debes recordar

  1. Las tareas programadas son agentes autonomos con horario fijo
  2. Necesitan el ordenador encendido, pero se recuperan si estaba apagado
  3. Consumen tokens — son agentes reales, no scripts
  4. Hooks no consumen tokens necesariamente — son automatizaciones ligeras
  5. Skills + Hooks + Tareas = sistema operativo completo para tu negocio

Tarea practica del modulo

  1. Abre la seccion "Programar" de Claude Code
  2. Usa el Prompt 1 para identificar 3 tareas automatizables
  3. Crea tu primera tarea programada con el Prompt 3 (resumen semanal)
  4. Ejecutala manualmente para verificar que funciona
  5. Cuando estes satisfecho, activa el modo bypass
